Deep Learning 学习日记

发表于 2017-04-04

学习心得

卷积神经网络

1.-卷积层：卷积的意义在于降维，降低参数的维度。减小模型的数量级

2.池化层：下采样，卷积之后得到特征，选择特征最明显的

3.激活层： 筛选得到的信息，通过激活函数选择比较合适的信息

4.全连接层： 两层之间所有神经元都有权重链接，通常在尾部

5.数据输入层： 去均值，归一化等

6.Dropout: 防止过拟合

学习总结

网络

  1. Lenet
  2. Alexnet
  3. ZFnet
  4. VGG
  5. GoogLeNet
  6. ResNet

框架：

1.caffe

  1. faster-rcnn
  2. fast-rcnn

RCNN->Fast-RCNN->Faster-RCNN

RCNN

  1. 提取特佂–在原图基础上用ss提取2000个特征框，然后将这2000个框送入卷积层，进行卷积计算，提取特征
  2. 分类–svm分类器，训练svm模型
  3. 使用回归器精细修正候选框位置

Fast-RCNN

  1. 提取特征–在rcnn的基础上进行改进，不是将2000个特征框送入卷积层进行特征提取，而是直接将整张图送入卷积网络，然后根据ss提取的2000个特征框，在原图卷积后得到的特征图上进行映射，直接提取2000个特征，大大提高了计算速度
  2. 分类–网络参数训练，网络直接输出各类概率(softmax)，比SVM分类器性能略好
  3. 调整位置

Faster-RCNN

  1. 提取特征–在fast-rcnn基础上，吸取其提取特征慢的教训，进一步提升速度，做到实时检测，提出一个新东西叫RPN，和分类器交替训练，RPN用于提取特征，不在卷积层外提取框在做映射，而是直接在原图得到的feature map 基础上进行特征提取，省去了前期提取特征框的时间，大大加快速度，使所有操作都在网络内完成，实现端到端
  2. 分类–网络参数训练，将分类的网络和RPN网络交替训练，最终直接得到分类结果
  3. 调整位置

Qt 踩坑记

发表于 2017-04-04

Qt 开发–踩坑记

1.Thread线程的使用注意套路，套路很简单

  1. 继承之后，实现自己的线程类，重写run函数
  2. 在主界面开启线程时直接调用start（）函数会自动调用run函数，run注意小写，注意小写，注意小写
  3. 如果调用线程类中其他公有函数，则相当于在当前线程执行，并不会重新开启一个线程在后台运行，该点一定注意
  4. 开启线程时，在qt creatorIDE下debug模式不能运行，必须是release模式才可以，但是vs装了Qt插件之后就可以调试线程了 (本人vs2015)

2.Qt信号槽机制真的是神器，可以实现线程通信，跨线程也可以使用，所以非常好,有关信号槽的理解与使用，推荐两篇博客Qt实现多线程下的信号与通讯和信号槽机制浅析

其中关于信号槽的的理解非常有帮助

3.Qt摄像头有好几种方法，见下：

  • 使用phonon，不过据说Qt5对其已经不太支持

  • 使用Qt自带的MultiMedia模块

  • 使用openCV

    openCV乃真神器，使用方便，只需在pro文件中引入openCV的头文件和lib文件即可

4.Qt在label上显示图片，并在图片上画框

这个东西困扰自己两天，究其原因不过是对Qt的不了解，其中的事件机制，自定义控件的方法，继承控件，实现自己功能并在ui中使用自己修改过的控件的方法等等

  1. 首先Qt的事件机制需要好好了解，推荐几篇博客吧事件处理机制以及QLabel的交互实现和Qt中事件处理的方法
  2. 引用最后帮我解决问题的一本书上的一句话：在Qt设计师中使用自定义的窗口部件，我们必须让Qt设计师先察觉到它的存在，有两种方法可以玩策划能这一任务:改进法(promotion)和插件法(plugin)(这个地方着实是令我困惑最近的地方)
  3. 一篇非常好的博客，虽然讲的不详细，但是对的qt中在QLabel上显示图片，并用鼠标在上面画框
